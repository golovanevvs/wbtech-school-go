# calendar

- путь до проекта внутри репозитория: `L4/L4.3/calendar`.

## Составные части

- api-сервер: `calendar_main-server`;
- web-клиент: `calendar_web-client`.

### calendar_main-server

- путь: `L4/L4.3/calendar/calendar_main-server`
- язык программирования: `Go`;
- БД: `PostgreSQL`:
  - путь к файлам миграции: `providers/migrate/migrations`;
- роутер: `Gin`;

### calendar_web-client

- путь: `L4/L4.3/calendar/calendar_web-client`;

- общее:
  - используемые основные фреймфорки и библиотеки: `next.js`, `material-ui`, `material-react-table`, `fullcalendar`;
  - для запуска команд вместо команды `npm` использовать команду `pnpm`;
  - после deploy клиент должен корректно работать через `nginx`, т.е. необходимо предусмотреть правильные пути для страниц с использованием `process.env.NEXT_PUBLIC_BASE_PATH`;
  - дизайн должен быть адаптирован в том числе к размерам экранов мобильных устройств.

- Header общий для всех страниц:
  - название "Calendar" -> переход на страницу "Главная" (`/`);
  - кнопка "Сменить тему"(`Brightness7Icon`, `Brightness4Icon`) -> переключение темы (светлая/тёмная);

- Footer общий для всех страниц:
  - текст, содержащий контакты разработчика (Telegram username: `@golovanevvs`);

- страница "Главная" (`/`):
  - загрузка данных о событиях на месяц с сервера (`GET /events/month`).
  - кнопка "Добавить" -> переход на страницу "Добавить событие" (`/event/create`);
  - календарь `fullcalendar` в режиме отображения событий на текущий месяц.

- страница "Добавить событие" (`/event/create`):
  - форма для ввода данных о событии.
  - кнопка "Создать" -> добавление события в базу данных (`POST /event/create`);
  - кнопка "Отмена" -> переход на страницу "Главная" (`/`).
