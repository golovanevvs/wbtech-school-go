# warehouse-control

## План реализация

- warehouse-control_main-server - api-сервер на Go;
- warehouse-control_web-client - web-клиент на next.js (для запуска команд вместо npm использовать pnpm).

### web-клиент (next.js)

- авторизация:
  - осуществляется с помощью access и refresh JWT (JWTs хранить в cookie);
  - три уровня доступа:
    - "Кладовщик";
    - "Менеджер";
    - "Аудитор";

- Header общий для всех страниц:
  - если авторизован:
    - кнопка "Меню":
      - кнопка меню "Список товаров" (отображается для уровней доступа "Кладовщик", "Менеджер") -> переход на страницу "Список товаров" (`/items`);
      - кнопка меню "История действий" -> переход на страницу "История действий" (`/history`);
    - название "Warehouse Control" -> переход на страницу "Главная" (`/`);
    - кнопка "Профиль" -> всплывающее меню:
      - кнопка меню "Профиль" -> переход на страницу "Профиль" (`/profile`);
      - кнопка меню "Сменить тему" -> переключение темы (светлая/тёмная);
      - кнопка меню "Выйти" -> выход из аккаунта, переход на страницу "Вход" (`/auth`);
  - если не авторизован:
    - название "Warehouse Control" -> переход на страницу "Главная" (`/`);

- Footer общий для всех страниц:
  - текст, содержащий контакты разработчика (Telegram username);

- страница "Главная" (`/`):
  - если авторизован:
    - кнопка "Список товаров" (отображается для уровней доступа "Кладовщик", "Менеджер") -> переход на страницу "Список товаров" (`/items`);
    - кнопка "История дествий" -> переход на страницу "История действий" (`/history`);
  если не авторизован:
    - автоматический переход на страницу "Вход" (`/auth`);
