# shortener_main-server

## Задачи

1 HTTP API
Обрабатывает запросы от UI и других клиентов.

- `/shorten` — создание короткой ссылки.
- `/s/{short}` — редирект на оригинальный URL.
- `/analytics/{short}` — получение аналитики.

2 Работа с БД

Сохраняет/читает `short_url` и `analytic` в/из PostgreSQL.
Использует репозитории, сервисы, модели.

3 Логика сокращения
Генерирует короткий код (если не задан).
Проверяет уникальность.
Сохраняет оригинальный URL.

4 Сохранение аналитики
При переходе — записывает событие в БД.

5 Интеграции
Подключение к PostgreSQL.
(В будущем) подключение к Redis (для кэширования).
Логирование, мониторинг (Prometheus).
